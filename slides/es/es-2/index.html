    Desarrollar con el modelo de datos de negocio

    • Relaciones : composición y agregación
    • Restricciones únicas
    • Consultas: por defecto o personalizadas
    • Índices
    • Accesible mediante la API REST

    /!\ Nunca hay que leer o escribir directamente desde la base de datos BDM /!\

    Persistencia gestionada por el motor Bonita a través de las bibliotecas Hibernate y la base de datos

    Business variable used BDM definition to define the type of the variable.
    Several solutions exist to Create (INSERT) / Read (SELECT) / Update (UPDATE) / Delete (DELETE).
    Business variables lifecycle is independent of the lifecycle of process instance. The data can be updated by several processes.
    Accessible by REST APIs if integration is needed with third party application or for application pages.
    Never try to read or write data directly from the BDM database, use the APIs to do so.
    [Studio - show data in the process to execute the flow]

    Desarrollar con el modelo de datos de negocio

    • 2 formas de cargar los datos: eager o lazy

    API/bdm/businessData/com.training.model.Employee/{persistenceId}

    Complex type with fields. Structured objects composed of: Java classes (Integer, Boolean, String, Date...) and other BDMs
    Java POJO generated out of the model.
    Stored in a dedicated database with automatic administration (tables creation, updates...).
    Persistence handle by Bonita using Hibernate library and the database.
    Transactional database storage guarantees that data is safely kept in case of server shut down (expected or not).

    Vista global de la arquitectura

    Envío de datos del formulario a la variable de negocio

    2.30

    When the user inputs some data in a form they are first stored in JavaScript variables.
    When the user clicks on the submit button, a REST API call is performed to the Engine API to instruct the Engine to start a new process instance or execute a task. This call carry the value of a JavaScript variable associated with the submit button. REST API call is an HTTP request that carry out data as JSON.
    Engine will check that received JSON data match the contract definition and validation rules. If it match, the data are now stored in the contract variables.
    Contract variables can then be used in various place such as a business variable default value (this will create a new row in business database), operation on a task to change the attribute value of a business variable (this will perform an UPDATE SQL request on the business database)...

    Otro tipo de datos: documentos

    • Bonita proporciona un sistema de gestión de documentos simple
    • Las referencias a documentos hacen parte de la definición del proceso
    • En tiempo de ejecución los documentos se pueden inicializar, actualizar ...
    • A diferencia de la BDM, los documentos están asociados con una instancia de proceso determinada

    Páginas

    Construida de la misma manera que los formularios
    Diferencia principal en el contexto de ejecución:

    • Los formularios tienen información sobre la definición del proceso o sobre la instancia del proceso y la tarea
    • Las páginas no tienen información de contexto

    Las páginas pueden obtener información del BDM o de los procesos o activar acciones a través de las API REST

    Agrupar varias páginas y un menú para crear una aplicación

    4

    Display information (BDM, process definitions, process instances) and optionally provide access to actions (start a process instance, execute a task...)
    When you create a page you don't have any variable available by default.
    Example to list vacation requests: http://localhost:8080/bonita/API/bdm/businessData/com.company.model.VacationRequest?q=find&p=0&c=10

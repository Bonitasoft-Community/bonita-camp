Modéliser les données : le Modèle de Données métier (BDM)

  • Stocke les données métier en lien avec les processus et/ou les applications
  • Définit des objets composé d'attributs
  • Editeur de Modèle de Données Métier dans Bonita Studio
  • Depuis ce modèle, Bonita génère des classes Java et crée les tables en base de données
  • La persistance est gérée par Bonita Engine, le moteur BPM de Bonita
On peut créer des objets complexes, en relation avec d'autres objets. Les objets peuvent donc être composés de classes Java (Integer, Boolean, String, Date...) et d'autres objets
Les classes Java (POJO) sont générées hors du modèle.
Le modèle est stocké dans une base de données dédiée, administrée de façon automatique (création des tables, mise à jour ...).
La persistence est gérées par Bonita en utilisant la library Hibernate.
Le stockage dans une base de données transactionnelle garantit la sécurité de la donnée en cas de redémarrage ou de panne serveur.

Utilisation du Modèle de Données Métier (BDM)

  • Déclarer des variables métier dans la définition du processus
  • Create/Read/Update/Delete en utilisant : valeur par défaut, opérations, connecteurs...
  • Peuvent être manipulées par différentes instances de différents processus
  • Cycle de vie contrôlé par le développeur
Le type de la variable métier est calqué sur le type de l'objet métier sur lequel elle est créée.
Plusieurs solutions existent pour Create (INSERT) / Read (SELECT) / Update (UPDATE) / Delete (DELETE).
Le cycle de vie des variables métier est indépendent de celui des instances de processus.
Elles sont accessible par BDM REST APIs, utile s'il est nécessaire de aire une intégration avec une application third party, ou y accéder depuis des pages d'applications.
Les processus sont les seuls à pouvoir mettre à jour la donnée. N'essayez pas de modifier directement en base de données.
Pour lire les informations en base de données, utilisez les API. [Studio - montrer où tout cela se passe]

2.30

Quand l'utilisateur insère une information dans un formulaire, elle est d'abord stockées dans une variable JavaScript.
Quand l'utilisateur clique sur le bouton "Soumettre", un appel REST API est fait vers Bonita Engine API pour demander à au moteur BPM de démarrer une nouvelle instance ou exécuter une tâche.
Cet appel transporte la valeur de la variable JS. L'appel REST API est une requête HTTP qui transporte l'information sous forme d'un JSON.
Bonita Engine vérifie que l'information JSON reçue correspond au contrat et aux règles de validation définies. En cas de correspondance, l'information est stockée dans des variables de contrat.
Les variables de contrat peuvent alors être utilisées à différents endroits (instancier une variable métier, changer la valeur d'une variable par une opération) et modifier ainsi la base de données...

Acteurs

  • Déclarer un ou plusieurs acteurs dans la définition du processus (juste un nom)
  • À configurer pour associer l'acteur aux utilisateurs :
    • Directement à un ensemble d'utilisateurs individuels
    • Indirectement via des groupes et des rôles

Les changements de l'organisation impactent qui peut faire quoi. Utiliser les acteurs rend les tâches tout de suite accessible à qui rejoint un groupe ou un rôle.
